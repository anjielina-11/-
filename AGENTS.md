# AGENTS.md — Codex 项目协作规范

## 基本原则

- 始终用中文沟通；技术术语、命令和代码标识符保留英文。
- 先理解项目结构、现有实现和依赖，再修改代码；不臆测技术栈或业务规则。
- 需求、账号、凭证、目标环境或方案选择存在关键歧义时，先说明问题并向用户确认。
- 只改与当前需求直接相关的内容；不顺手重构、格式化或删除无关代码。
- 每次完成修改后，简要说明改动内容、原因和验证结果。

## 工作方式

- 优先使用项目已有的工具链、代码风格和包管理器：`package-lock.json` 使用 `npm`，`pnpm-lock.yaml` 使用 `pnpm`，`yarn.lock` 使用 `yarn`。
- 查找内容时先使用 `rg` 或文件列表定位，再读取必要文件；独立检查可并行执行。
- 不读取或修改 `.codeartsdoer/`、`.idea/`、`node_modules/`、`__pycache__/`、`output/`。
- 新建文件前确认其不被 `.gitignore` 忽略；删除文件前确认没有引用。
- 图片、截图和文档中的图片引用属于上下文的一部分，必须先查看再据此处理。

## 代码约定

### Python

- 遵循 PEP 8，使用 4 空格缩进。
- 网络请求应有合理的超时、异常处理；文件操作使用 `with`。
- 爬虫设置合理的 `DOWNLOAD_DELAY` 和 User-Agent，依赖写入 `requirements.txt`。

### JavaScript / TypeScript

- 使用 2 空格缩进、单引号、无分号；使用 `const`/`let`，禁止 `var`。
- 异步流程优先使用 `async/await`。
- 不擅自关闭 TypeScript `strict`；交付前运行 `tsc --noEmit`（适用时）。

### Vue

- 使用 `<script setup lang="ts">`；组件顺序为 `<template>`、`<script>`、`<style>`。
- Props 和 emits 必须声明 TypeScript 类型；状态管理优先 Pinia，路由使用 Vue Router。

## 新建项目

- 新建项目时先确认技术栈、包管理器、运行方式和验证命令，再开始实现。
- 各子项目独立管理依赖、配置和构建产物，禁止跨项目引用或混用依赖。

## 交付前检查

- 按改动范围运行可用的 test、lint、build 或类型检查；不虚报未执行的验证。
- 核对需求逐项覆盖，并检查空数据、网络错误、特殊字符与未登录等适用边界。
- Node.js 服务确认可启动；Python 脚本确认无 import 错误；爬虫确认能正确产出数据。

## 交付格式

```text
✅ 完成
📋 验证: 已执行的测试 / 类型检查 / 功能检查
📁 改动文件: ...
💡 备注: 风险、未执行项或后续操作（如有）
```
