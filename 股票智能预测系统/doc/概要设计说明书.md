# 股票智能预测系统 — 概要设计说明书

> 版本：v1.0 | 日期：2026-07-19 | 作者：开发团队

---

## 1. 引言

### 1.1 编写目的

本文档描述股票智能预测系统的总体架构设计、模块划分、技术选型和接口设计，作为详细设计及编码实现的纲领性文件。

### 1.2 设计原则

- **模块化**：前端、后端、算法三大模块独立，可并行开发
- **低耦合**：模块间通过 REST API 通信
- **可扩展**：算法模块可独立升级，不影响前后端
- **配置集中**：所有配置集中于 `config.py`

## 3. 技术选型

| 层级 | 技术 | 版本 | 选型理由 |
|------|------|------|----------|
| 前端框架 | 原生 HTML/CSS/JS | — | 轻量，无构建工具依赖 |
| 图表库 | ECharts 5.6 | CDN | 国产、中文支持好、主题可定制 |
| 后端框架 | Flask 3.x | Python | 轻量、灵活、学习成本低 |
| 数据获取 | akshare | 最新 | 免费、无需 token、覆盖 A 股全量 |
| 深度学习 | PyTorch 2.x | Python | 工业标准、Transformer 支持完善 |
| 大模型 | DeepSeek API | V3 | 国产、中文能力强、性价比高 |
| 数据库 | SQLite 3 | 内置 | 零配置、嵌入式、适合单用户场景 |
| 科学计算 | NumPy / Pandas | 最新 | 数据处理事实标准 |
| 数据预处理 | scikit-learn | 最新 | MinMaxScaler 归一化 |

## 4. 模块设计

### 4.1 前端模块 (Frontend)

| 文件 | 职责 |
|------|------|
| `index.html` | 股票搜索、预测触发、ECharts 图表渲染、AI 分析展示、模型信息展示、导航栏 |
| `favorites.html` | 收藏列表管理（增/删）、搜索添加、一键分析跳转 |

**数据流**：用户操作 → fetch() 调用后端 API → 解析 JSON → 渲染图表/表格/文本

### 4.2 后端模块 (Backend)

| 文件 | 职责 |
|------|------|
| `__init__.py` | Flask 应用工厂 (`create_app`)、SQLite 初始化、请求级数据库连接管理 |
| `config.py` | 集中管理数据库路径、DeepSeek Key、Flask 配置、初始收藏列表 |
| `routes.py` | Blueprint 路由：页面路由 + 6 个 REST API 端点 |
| `stock_service.py` | akshare 封装：股票列表、名称搜索、日线数据获取 |

**API 端点清单**：

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/` | 主页页面 |
| GET | `/favorites` | 收藏页面 |
| POST | `/api/search` | 股票名称模糊搜索 |
| POST | `/api/predict` | Transformer 预测 |
| POST | `/api/analyze` | DeepSeek AI 分析 |
| GET | `/api/favorites` | 获取收藏列表 |
| POST | `/api/favorites` | 添加收藏 |
| DELETE | `/api/favorites/<id>` | 删除收藏 |

### 4.3 算法模块 (Algorithm)

| 文件 | 职责 |
|------|------|
| `model.py` | `PositionalEncoding`、`TimeSeriesTransformer` 模型定义、`StockPredictor` 训练/预测封装 |

**核心类关系**：

```
StockPredictor
├── model: TimeSeriesTransformer
│   ├── input_proj: Linear(1→64)
│   ├── pos_encoder: PositionalEncoding
│   ├── transformer_encoder: TransformerEncoder × 3
│   └── output_proj: Sequential(1280→128→5)
├── scaler: MinMaxScaler(-1, 1)
├── train(prices) → dict (含 train_losses/val_losses/模型元数据)
└── predict(last_20_prices) → ndarray[5]
```

## 6. 接口设计

### 6.1 前后端接口

所有 API 采用 JSON 格式通信，编码 UTF-8。

#### POST /api/predict

**请求**：
```json
{ "stock_name": "平安银行" }
```

**响应**：
```json
{
  "stock_name": "平安银行",
  "stock_code": "000001",
  "historical": [{ "date": "20260701", "close": 10.50 }, ...],
  "predicted": [{ "date": "20260720", "close": 10.74 }, ...],
  "model_info": {
    "architecture": { "d_model": 64, "attention_heads": 4, ... },
    "training_config": { "optimizer": "Adam", ... },
    "complexity": { "total_params": 314693, ... },
    "data_stats": { "total_samples": 476, ... },
    "metrics": { "final_train_loss": 0.017, ... }
  },
  "training_info": {
    "train_losses": [0.12, 0.08, ...],
    "val_losses": [0.15, 0.10, ...]
  }
}
```

### 6.2 外部接口

#### DeepSeek Chat API

- **端点**：`POST https://api.deepseek.com/v1/chat/completions`
- **认证**：Bearer Token
- **模型**：`deepseek-chat`
- **参数**：temperature=0.7, max_tokens=2048

#### akshare 数据接口

- `ak.stock_info_a_code_name()` — 获取 A 股全量列表
- `ak.stock_zh_a_daily(symbol, adjust)` — 获取日线数据
