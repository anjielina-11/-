# 股票智能预测系统 — 需求分析说明书

> 版本：v1.0 | 日期：2026-07-19 | 作者：开发团队

---

## 1. 引言

### 1.1 编写目的

本文档旨在明确"股票智能预测系统"的功能需求、性能需求及用户交互需求，作为后续概要设计、详细设计、编码实现和测试验收的依据。

### 1.2 项目背景

随着人工智能技术在金融领域的广泛应用，基于深度学习的股价预测成为量化投资的重要工具。本项目面向个人投资者，提供一套基于 Transformer 深度学习模型的 A 股短期价格预测系统，辅助用户进行投资决策。

### 1.3 术语定义

| 术语 | 说明 |
|------|------|
| A股 | 在中国境内注册、在沪深交易所上市的人民币普通股票 |
| Transformer | 一种基于自注意力机制的深度学习架构，原用于NLP，本项目用于时间序列预测 |
| Tushare / akshare | 中国股票行情数据的开源数据源 |
| DeepSeek | 深度求索公司提供的大语言模型 API |

---

## 2. 系统概述

### 2.1 系统目标

构建一个 Web 应用，允许用户输入 A 股股票名称，系统自动获取历史数据，使用 Transformer 模型预测未来 5 个交易日收盘价，并通过 DeepSeek 大模型生成投资分析报告。

### 2.2 用户角色

| 角色 | 描述 |
|------|------|
| 普通用户 | 搜索股票、查看预测结果、阅读 AI 分析报告、管理收藏列表 |

---

## 3. 功能需求

### 3.1 功能模块划分

#### FR-01：股票搜索模块

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-01-01 | 用户输入中文股票名称进行模糊搜索 | P0 |
| FR-01-02 | 系统自动将股票名称转换为对应的 6 位股票代码 | P0 |
| FR-01-03 | 搜索结果以下拉列表形式展示，包含名称、代码、交易所 | P0 |

#### FR-02：数据获取模块

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-02-01 | 以系统当前日期为基准，获取最近 500 个交易日收盘价 | P0 |
| FR-02-02 | 获取最近 20 个交易日收盘价作为预测输入 | P0 |
| FR-02-03 | 支持沪深两市所有正常上市 A 股 | P0 |

#### FR-03：预测模型模块

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-03-01 | 使用 Transformer Encoder 架构进行时间序列预测 | P0 |
| FR-03-02 | 基于历史 20 个交易日收盘价，预测未来 5 个交易日收盘价 | P0 |
| FR-03-03 | 展示模型架构参数、训练配置、损失曲线等业界标准指标 | P1 |

#### FR-04：AI 分析模块

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-04-01 | 调用 DeepSeek 大模型 API 生成投资分析报告 | P1 |
| FR-04-02 | 分析内容包含：经营业务概述、财务数据、投资建议（含风险提示） | P1 |

#### FR-05：收藏管理模块

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-05-01 | 展示收藏股票列表，初始包含 5 支预设 A 股 | P1 |
| FR-05-02 | 支持添加/删除收藏股票 | P1 |
| FR-05-03 | 每支股票提供"分析"按钮，点击跳转主页并自动触发预测 | P1 |
| FR-05-04 | 收藏数据持久化存储在数据库中 | P0 |

#### FR-06：可视化展示模块

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-06-01 | 以折线图展示历史收盘价与预测收盘价 | P0 |
| FR-06-02 | 日期格式为"xx月xx日"（如"04月24日"） | P0 |
| FR-06-03 | 历史数据为蓝色实线，预测数据为紫色虚线，标注预测起点 | P0 |
| FR-06-04 | 以表格形式展示预测详情（日期、价格、涨跌幅） | P1 |
| FR-06-05 | 以折线图展示模型训练损失曲线 | P1 |

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 预测响应时间 | 单次预测（含数据获取+模型训练）< 60 秒 |
| AI 分析响应时间 | < 30 秒 |
| 页面首次加载 | < 3 秒 |
| 并发支持 | 开发环境单用户 |

### 4.2 可用性需求

- 界面采用深色主题，视觉统一
- 支持主流浏览器（Chrome、Edge、Firefox）
- 中文界面，专业术语需通俗化解释

### 4.3 可维护性需求

- 项目按前端/后端/算法三模块分离
- 配置集中管理，不硬编码
- 提供完整的项目文档

### 4.4 约束条件

- 数据源依赖第三方 API（akshare），需网络连接
- AI 分析依赖 DeepSeek API，需有效 API Key
- 模型训练依赖 CPU/GPU 计算资源

## 5. 数据需求

### 5.1 输入数据

| 数据项 | 格式 | 来源 |
|--------|------|------|
| 股票名称 | 中文字符串 | 用户输入 |
| 历史日线数据 | 日期/开/高/低/收/量 | akshare API |

### 5.2 输出数据

| 数据项 | 格式 | 说明 |
|--------|------|------|
| 预测收盘价 | 浮点数 (2位小数) | 未来5个交易日 |
| AI 分析报告 | Markdown 文本 | 含三大板块 |
| 模型元数据 | JSON | 架构/训练/复杂度/数据统计 |

### 5.3 持久化数据

| 数据 | 存储方式 | 说明 |
|------|----------|------|
| 收藏股票列表 | SQLite | favorites 表 |

---

## 6. 验收标准

1. 用户输入"平安银行"后能正确识别代码 000001 并获取数据
2. 预测流程在 60 秒内完成并返回 5 个预测值
3. 图表正确显示历史数据和预测数据的分界线
4. AI 分析报告含经营业务、财务数据、投资建议三部分
5. 收藏列表可正常增删，刷新页面后数据不丢失
6. 收藏页"分析"按钮能正确跳转主页并自动触发预测
