## 2. 算法模块详细设计

### 2.1 文件：`algorithm/model.py`

### 2.1.1 类：`PositionalEncoding`

继承 `nn.Module`，提供正弦/余弦位置编码。

```
PositionalEncoding
├── __init__(d_model=64, dropout=0.1, max_len=500)
│   计算 pe[position, 2i]   = sin(pos / 10000^(2i/d_model))
│   计算 pe[position, 2i+1] = cos(pos / 10000^(2i/d_model))
│   注册为 buffer（不参与梯度计算）
└── forward(x: [B, L, D]) → [B, L, D]
    x = x + pe[:, :L, :]
    return dropout(x)
```

### 2.1.2 类：`TimeSeriesTransformer`

```
TimeSeriesTransformer(nn.Module)
├── 属性
│   ├── input_len: int = 20
│   ├── output_len: int = 5
│   ├── d_model: int = 64
│   ├── input_proj: nn.Linear(1, 64)
│   ├── pos_encoder: PositionalEncoding(64, 0.1)
│   ├── transformer_encoder: nn.TransformerEncoder
│   │   └── encoder_layer: nn.TransformerEncoderLayer
│   │       ├── self_attn: MultiheadAttention(nhead=4)
│   │       ├── linear1: Linear(64, 256)
│   │       ├── dropout: Dropout(0.1)
│   │       ├── linear2: Linear(256, 64)
│   │       ├── norm1: LayerNorm(64)
│   │       ├── norm2: LayerNorm(64)
│   │       └── dropout1/2: Dropout(0.1)
│   └── output_proj: nn.Sequential
│       ├── Linear(64*20, 128)
│       ├── ReLU()
│       ├── Dropout(0.1)
│       └── Linear(128, 5)
└── forward(x: [B, 20, 1]) → [B, 5]
    x = input_proj(x)          → [B, 20, 64]
    x = pos_encoder(x)         → [B, 20, 64]
    x = transformer_encoder(x) → [B, 20, 64]
    x = x.reshape(B, -1)       → [B, 1280]
    return output_proj(x)      → [B, 5]
```

### 2.1.3 类：`StockPredictor`

```
StockPredictor
├── 属性
│   ├── device: torch.device
│   ├── model: TimeSeriesTransformer | None
│   ├── scaler: MinMaxScaler | None
│   ├── input_len: int = 20
│   ├── output_len: int = 5
│   └── model_config: dict (d_model/nhead/layers/d_ff/dropout)
│
├── _count_parameters() → dict
│   返回 {"total": N, "trainable": M}
│
├── _create_sequences(data: ndarray) → (X, y)
│   滑动窗口: input_len=20, output_len=5
│   输入: 归一化后的价格数组 [n]
│   输出: X [n-len-5+1, 20], y [n-len-5+1, 5]
│
├── train(prices, epochs=100, lr=0.001, verbose=True) → dict
│   ├── 1. MinMaxScaler(-1,1) 拟合归一化
│   ├── 2. 创建训练样本 (X, y)
│   ├── 3. 80/20 划分训练/验证集
│   ├── 4. 初始化 TimeSeriesTransformer → device
│   ├── 5. 训练循环 (epoch=0..99):
│   │   ├── forward → loss (MSE)
│   │   ├── backward + gradient_clip(1.0)
│   │   ├── validation → val_loss
│   │   └── scheduler.step(val_loss) [ReduceLROnPlateau]
│   └── 6. 返回: {
│       train_losses, val_losses,
│       final_train_loss, final_val_loss,
│       train_samples, val_samples, total_samples,
│       epochs, train_time_sec, device,
│       model_config, input_len, output_len,
│       param_count, optimizer, learning_rate,
│       loss_function, scheduler, gradient_clip_norm,
│       scaler, train_val_split
│   }
│
└── predict(last_prices: ndarray[20]) → ndarray[5]
    ├── scaler.transform() 归一化
    ├── model.forward() 预测 (no_grad)
    └── scaler.inverse_transform() 反归一化 → 返回真实价格
```

### 3.3 文件：`backend/routes.py`

Blueprint 名称：`api`，包含以下路由处理函数：

| 函数 | 方法 | 路径 | 输入 | 输出 |
|------|------|------|------|------|
| `index()` | GET | `/` | — | HTML |
| `favorites_page()` | GET | `/favorites` | — | HTML |
| `search_stock()` | POST | `/api/search` | `{keyword}` | `{stocks: [...]}` |
| `predict()` | POST | `/api/predict` | `{stock_name}` | `{stock_name, stock_code, historical, predicted, model_info, training_info}` |
| `analyze()` | POST | `/api/analyze` | `{stock_name, stock_code, predicted, historical}` | `{analysis}` |
| `list_favorites()` | GET | `/api/favorites` | — | `{favorites: [...]}` |
| `add_favorite()` | POST | `/api/favorites` | `{stock_name, stock_code}` | `{success, message}` |
| `remove_favorite()` | DELETE | `/api/favorites/<id>` | — | `{success, message}` |

**predict() 函数处理流程**：

```
1. 解析 JSON → stock_name
2. stock_service.get_stock_code_by_name(name) → code
   ├── code 为空 → 404
3. stock_service.get_training_data(code, 500) → prices, dates
   ├── 数据 < 50 条 → 400
4. StockPredictor().train(prices) → training_result
5. stock_service.get_recent_close_prices(code, 20) → recent
6. predictor.predict(recent) → predictions[5]
7. 计算未来5个交易日日期（跳过周末）
8. 构建 model_info 字典（5个子类别）
9. 构建响应 JSON → 返回
10. 异常处理: ValueError→400, RuntimeError→500, Except→500
```

### 3.4 文件：`backend/stock_service.py`

对外暴露的函数：

| 函数 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `get_stock_list()` | — | `DataFrame` | 获取 A 股全量列表（7天缓存） |
| `search_stock_by_name(keyword)` | str | `list[dict]` | 名称模糊搜索（最多20条） |
| `get_stock_code_by_name(name)` | str | `str|None` | 精确匹配名称→代码 |
| `get_daily_data(code, limit)` | str, int | `DataFrame` | 获取日线数据（含日期格式化） |
| `get_recent_close_prices(code, days)` | str, int | `(ndarray, list)` | 最近 N 日收盘价+日期 |
| `get_training_data(code, days)` | str, int | `(ndarray, list)` | 获取训练用大量历史数据 |

**数据获取流程**：

```
股票代码 "000001"
  → _code_to_ak_symbol("000001") → "sz000001"
  → ak.stock_zh_a_daily(symbol="sz000001", adjust="qfq")
  → 重命名 date→trade_date，格式化日期 YYYYMMDD
  → 按日期升序排列，取最近 N 条
  → 返回 DataFrame
```

```
.container
├── .header (标题 + 副标题)
```

**关键 JavaScript 函数**：

| 函数 | 说明 |
|------|------|

| `startPrediction()` | 调用 `/api/predict`，触发所有渲染 |
| `renderMainChart(data)` | 渲染 ECharts 折线图（历史蓝实线 + 预测紫虚线） |

| `init()` | 页面加载时：获取收藏数量、检查 `?analyze=` 参数自动触发 |

**自动分析机制**：

```javascript
// URL: /?analyze=平安银行
const params = new URLSearchParams(window.location.search);
const analyzeStock = params.get('analyze');
if (analyzeStock) {
    stockInput.value = decodeURIComponent(analyzeStock);
    window.history.replaceState({}, document.title, '/');  // 清理URL
    startPrediction();  // 自动触发预测+AI分析
}
```

## 5. 数据流

### 5.1 完整预测流程数据流

```
用户输入 "平安银行"
  │
  ▼
POST /api/predict { stock_name: "平安银行" }
  │
  ├─► stock_service.get_stock_code_by_name("平安银行") → "000001"
  │
  ├─► stock_service.get_training_data("000001", 500)
  │     └─► ak.stock_zh_a_daily(symbol="sz000001") → prices[500], dates[500]
  │
  ├─► StockPredictor.train(prices)
  │     ├─► MinMaxScaler.fit_transform()
  │     ├─► _create_sequences() → X[476,20], y[476,5]
  │     ├─► 80/20 划分
  │     ├─► 训练 100 epochs
  │     └─► 返回 training_result
  │
  ├─► stock_service.get_recent_close_prices("000001", 20)
  │     └─► recent_prices[20]
  │
  ├─► predictor.predict(recent_prices)
  │     └─► future_prices[5]
  │
  └─► 响应 JSON (stock_name, predicted, model_info, training_info)
        │
        ▼
  前端渲染: 图表 + 表格 + 模型信息 + 损失曲线
        │
        ▼
  POST /api/analyze { stock_name, predicted, historical }
        │
        └─► DeepSeek API → analysis_text
              │
              ▼
        前端渲染: AI 分析报告 (Markdown→HTML)
```

## 6. 异常处理设计

| 异常类型 | HTTP 状态码 | 前端处理 |
|----------|-------------|----------|
| 股票名称不存在 | 404 | 红色错误提示 |
| 数据不足（< 50条） | 400 | 红色错误提示 |
| 搜索无结果 | 200 | 灰色"未找到"提示 |
| AI 分析超时 | 504 | "请求超时，请稍后重试" |
| AI 服务不可用 | 502 | "无法连接到 AI 分析服务" |
| 网络错误 | — | "请求失败: xxx" |
| 重复收藏 | 409 | "已在收藏列表中" |
| 重复删除 | 404 | "收藏记录不存在" |
