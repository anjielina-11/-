## 一、引言

### 1.1 项目背景

云南地处低纬高原，气候类型多样，特色农业（水稻、玉米、马铃薯、番茄、柑橘、茶叶等）是当地农民主要收入来源。然而病虫害频发且种类繁多，农户往往缺乏专业诊断能力，导致错过最佳防治时机。传统农技推广依赖基层农技人员下乡，存在覆盖面有限、响应慢、知识更新滞后等问题。

本项目利用深度学习图像识别 + RAG 检索增强 + AI Agent 决策编排技术，构建面向云南特色农业的智能诊断与生产管理平台，帮助农户快速准确识别病害、获取科学防治建议，并为农技人员、合作社管理人员提供数据驱动的生产管理工具。

### 1.2 项目范围

本系统覆盖以下核心能力：
- 作物病害图像智能诊断（18 类常见病害）
- 农技知识检索与防治建议生成（RAG + LLM）
- 农场/地块/种植周期管理
- 农事任务生成、分配与跟踪
- 天气与市场价格数据采集与趋势分析
- 模型性能监控（数据漂移、未知样本拒识）
- 多角色权限管理（农户/农技人员/合作社管理员/系统管理员）

### 1.3 术语定义

| 术语 | 说明 |
|------|------|
| RAG | Retrieval-Augmented Generation，检索增强生成，从知识库检索相关内容辅助 LLM 生成 |
| Agent | AI 智能体，编排推理流程（图像识别→RAG检索→天气分析→生成建议） |
| 拒识 | 对未知病害拒绝给出低置信度错误结果，转入人工审核 |
| 数据漂移 | 模型推理数据的分布随时间发生变化，可能导致准确率下降 |
| PostGIS | PostgreSQL 地理空间扩展，支持地图坐标存储 |
| pgvector | PostgreSQL 向量扩展，支持语义相似度搜索 |

## 二、用户故事

### 2.1 农户（Farmer）

| ID | 用户故事 | 验收标准 |
|----|----------|----------|
| US-F01 | 作为农户，我能注册/登录系统，管理我的个人信息 | 支持用户名+密码注册/登录，JWT 认证 |
| US-F02 | 作为农户，我能创建农场并添加地块，记录位置和面积 | 支持地图选点，面积单位(亩) |
| US-F03 | 作为农户，我能在地块上登记种植作物，记录播种日期和预期收获日 | 支持从作物库选择品种，自动计算生育期 |
| US-F04 | 作为农户，我能上传作物异常叶片/果实图片，获取 AI 病害诊断结果 | 支持拍照/相册上传，10 秒内返回病害名称+置信度 |
| US-F05 | 作为农户，我能查看 AI 诊断详情，包括防治建议和参考资料 | 展示病害分析→农业防治→物理防治→化学防治→注意事项 |

| US-F07 | 作为农户，我能完成任务并提交反馈，评价防治效果 | 支持文本反馈+星级评分 |
| US-F08 | 作为农户，我能查看自己农场的天气趋势和附近市场价格 | 7 天天气趋势图，30 天价格走势图 |

## 五、非功能需求

### 5.1 性能

| ID | 要求 | 指标 |
|----|------|------|
| NFR-P01 | 图片上传响应 | < 3 秒（不含 AI 推理） |
| NFR-P02 | AI 推理延迟 | < 15 秒（单张图片） |
| NFR-P03 | API 响应时间 | P95 < 500ms |
| NFR-P04 | 并发支持 | ≥ 50 并发用户 |
| NFR-P05 | 数据库查询 | 有索引的查询 < 100ms |

| ID | 要求 |
|----|------|

| NFR-S03 | CORS 白名单限制 |
| NFR-S04 | 文件上传大小限制（20MB）和类型白名单 |
| NFR-S05 | SQL 注入防护（MyBatis-Plus 参数化） |
| NFR-S06 | 软删除（所有业务表 deleted 字段） |
| NFR-S07 | 操作审计日志（AOP 切面） |
